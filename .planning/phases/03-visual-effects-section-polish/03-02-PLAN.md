  <files>src/styles/tailwind.css</files>
  <action>
Update the responsive font size tokens in `src/styles/tailwind.css` `@theme` block to create proportional steps down from the hero headline (8rem max).

**Current values and proposed changes:**

The hero is `clamp(3rem, 10vw, 8rem)`. Section titles need to be meaningfully larger than current `clamp(1.75rem, 3vw, 2.5rem)` to feel proportional to the bigger hero.

Update these tokens:

1. `--text-section-title`: `clamp(1.75rem, 3vw, 2.5rem)` → `clamp(2rem, 3.5vw, 3.25rem)`
   - Rationale: At desktop, 3.25rem (~52px) vs hero 8rem (128px) = ~2.5:1 ratio. Current 2.5rem (40px) feels too small relative to 128px hero.

2. `--text-cta-title`: `clamp(2rem, 4vw, 3rem)` → `clamp(2.25rem, 4.5vw, 3.75rem)`
   - Rationale: CTA is the closing bookend — should be slightly larger than section titles to draw attention. 3.75rem (60px) creates a 2.1:1 ratio with hero.

3. `--text-section-kicker`: `clamp(1.125rem, 1.5vw + 0.5rem, 1.625rem)` → `clamp(1.25rem, 1.5vw + 0.5rem, 1.75rem)`
   - Rationale: Slight bump to kicker (problem section closing line) keeps it proportional to larger section titles.

**Do NOT change:**
- `--text-hero` (already set correctly in Phase 1)
- `--text-hero-tagline` (already set correctly in Phase 1)
- `--text-hero-cta` (already set correctly in Phase 1)
- `--text-section-prose` (body text size is fine)
- Any fixed-size tokens (text-13, text-15, etc.)

**Hierarchy at desktop max sizes after changes:**
- Hero headline: 8rem (128px)
- CTA title: 3.75rem (60px)
- Section titles: 3.25rem (52px)
- Section kicker: 1.75rem (28px)
- Section prose: 1.3125rem (21px)
- Body: 1.125rem (18px)

This creates a clear descending scale: 128 → 60 → 52 → 28 → 21 → 18.
